Context

Chef's Touch was the main project for a design depth course, Technology, Accessibility, and Design. I took this class during the spring semester of my junior year at Olin College of Engineering.

Over the course of this project, we worked with Brian Charlson, Director of Technology at Carroll Center for the Blind, an amateur chef with a passion for accessible tools and trying new things. With his expert advice and co-design help, we worked towards implementing an accessible recipe repository that would work with screen readers.

Project Outcome

Chef's Touch is a functional web app, to which users can add recipes and search those added. Formatting that would be difficult for a screen reader (such as the unicode for one half) to parse is converted to a more understandable format in any occurance, and cooking tips are linked where they are most relevant. These features are in response to multiple design sessions we had with Brian Charlson.

Due to its nature as a community created and curated repository, Chef's Touch is still very small and has not undergone mass uptake. Several of the members on the team still hope to return to it and fix the remaining bugs.

My Role

As this class was a design depth, I took an equal part in coming up with the project and speaking with our primary user. I was also one of the two main technical leads, focused on the back end of the web app.

Working on the back end of this project, I primarily worked with accepting and processing user input to make it ready for display, then storing it in the database for later retrieval. Though I came into this project with only minor experience with queries and minimal web development experience, I took the primary role in constructing the search page, input processing, and database structure.

Primary Technologies

This project was written in Python, with a layer of HTML/CSS. We used Flask and Heroku to construct the website, and the recipes were stored in a mongoDB through MongoAlchemy.

Take-Aways

This project gave me a much firmer grasp on accessible design, as well as user-oriented design in general. It also helped me better understand how a web app functions and what would go into making more such projects in the future.
